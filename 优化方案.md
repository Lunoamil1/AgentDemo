## 四、P1 — 架构重构升级

### 4.1 懒加载与单例模式

**问题**：当前 `agent_tools.py` 模块级 `rag=RagSummarizeService()` 和 `model/factory.py` 模块级实例化，导致 import 即触发 FAISS 加载 + API 连接；且 `DashScopeEmbeddings` 被创建了 2 次。

**方案**：

```
改造前:
  agent_tools.py   →  rag = RagSummarizeService()       # import 时立即初始化
  model/factory.py →  chat_model = ChatModelFactory()()  # import 时立即初始化
  vector_store.py  →  DashScopeEmbeddings(...)           # 又创建一份

改造后:
  model/factory.py →  @lru_cache def get_chat_model()
                      @lru_cache def get_embeddings()     # 全局唯一单例
  agent_tools.py   →  def _get_rag_service():
                          if not hasattr(_get_rag_service, '_inst'):
                              _get_rag_service._inst = RagSummarizeService()
                          return _get_rag_service._inst   # 首次调用才初始化
  vector_store.py  →  使用 factory.get_embeddings()       # 复用全局单例
```

**收益**：启动速度提升、Embedding 实例从 2 降为 1、避免循环导入

### 4.2 配置管理统一化

**问题**：`config_handler.py` 4 个函数逻辑重复；`prompt_loader.py` 3 个函数逻辑重复；配置分散且无类型校验。

**方案**：

```
改造前:
  config_handler.py  →  load_rag_config / load_FAISS_config / load_prompts_config / load_agent_config  (4个重复函数)
  prompt_loader.py   →  load_system_prompt / load_rag_prompt / load_report_prompt  (3个重复函数)

改造后:
  config_handler.py  →  def load_config(name: str) -> dict
                        # name 映射: "rag" → config/rag.yml, "prompts" → config/prompts.yml ...
                        # 增加 yaml.safe_load 替代 yaml.load（安全性）
                        # 增加配置项类型校验（separators 必须为 list 等）

  prompt_loader.py   →  def load_prompt(key: str) -> str
                        # 统一: key → prompts_conf[key] → 读取文件内容
                        # 使用 with 语句，杜绝资源泄漏
```

**收益**：代码行数减半、新增配置零成本、类型安全

# tools/registry.py（新增）
from typing import Callable
from langchain_core.tools import BaseTool

class ToolRegistry:
    _tools: dict[str, BaseTool] = {}

## 五、P2 — 安全加固

### 5.1 API Key 管理

**问题**：当前依赖环境变量 `DASHSCOPE_API_KEY`，无校验、无加密、无审计。

**方案**：

```python
# utils/secret_manager.py（新增）
import os
from utils.logger_handler import logger

class SecretManager:
    REQUIRED_KEYS = ["DASHSCOPE_API_KEY"]

    @classmethod
    def validate(cls) -> bool:
        missing = [k for k in cls.REQUIRED_KEYS if not os.getenv(k)]
        if missing:
            logger.error(f"缺少必要环境变量: {missing}")
            raise EnvironmentError(f"缺少必要环境变量: {missing}")
        return True

    @classmethod
    def get(cls, key: str) -> str:
        value = os.getenv(key)
        if not value:
            raise EnvironmentError(f"环境变量 {key} 未设置")
        return value
```

```python
# app.py 启动前校验
from utils.secret_manager import SecretManager
SecretManager.validate()
```

**收益**：启动即校验、失败快速报错、避免运行时才发现密钥缺失

### 5.2 FAISS 安全反序列化

**问题**：`FAISS.load_local(allow_dangerous_deserialization=True)` 允许任意 pickle 反序列化，存在 RCE 风险。

**方案**：

```python
# 方案一：签名校验（推荐）
import hmac, hashlib

def save_with_signature(db, path):
    db.save_local(path)
    # 对 index.faiss 计算签名
    index_path = os.path.join(path, "index.faiss")
    with open(index_path, "rb") as f:
        sig = hmac.new(SECRET_KEY, f.read(), hashlib.sha256).hexdigest()
    with open(index_path + ".sig", "w") as f:
        f.write(sig)

def load_with_verify(path, embeddings):
    index_path = os.path.join(path, "index.faiss")
    sig_path = index_path + ".sig"
    if os.path.exists(sig_path):
        with open(index_path, "rb") as f:
            actual = hmac.new(SECRET_KEY, f.read(), hashlib.sha256).hexdigest()
        with open(sig_path) as f:
            expected = f.read().strip()
        if not hmac.compare_digest(actual, expected):
            raise SecurityError("FAISS 索引签名校验失败，文件可能被篡改")
    return FAISS.load_local(path, embeddings, allow_dangerous_deserialization=True)
```

**收益**：防止索引文件被恶意篡改后的反序列化攻击

### 5.3 用户输入校验

**问题**：用户输入直接传入 Agent 和 RAG 检索，无长度限制、无注入防护。

**方案**：

```python
# utils/input_validator.py（新增）
MAX_INPUT_LENGTH = 2000

def validate_user_input(text: str) -> str:
    if not text or not text.strip():
        raise ValueError("输入不能为空")
    if len(text) > MAX_INPUT_LENGTH:
        raise ValueError(f"输入长度不能超过 {MAX_INPUT_LENGTH} 字符")
    # 检测潜在 prompt injection
    injection_patterns = ["ignore previous", "disregard", "system:", "你是一个"]
    text_lower = text.lower()
    for pattern in injection_patterns:
        if pattern in text_lower:
            logger.warning(f"检测到潜在注入模式: {pattern}")
    return text.strip()
```

**收益**：防止超长输入导致 token 溢出、基础注入检测

### 6.1 结构化日志

**问题**：当前日志为纯文本格式，无法被 ELK / Loki 等日志平台解析。

**方案**：

```python
# utils/logger_handler.py 改造
import json

class StructuredFormatter(logging.Formatter):
    def format(self, record):
        log_data = {
            "timestamp": self.formatTime(record),
            "level": record.levelname,
            "logger": record.name,
            "message": record.getMessage(),
            "module": record.module,
            "line": record.lineno,
        }
        if hasattr(record, "extra_data"):
            log_data.update(record.extra_data)
        if record.exc_info:
            log_data["exception"] = self.formatException(record.exc_info)
        return json.dumps(log_data, ensure_ascii=False)
```

**收益**：日志可被日志平台索引、支持按字段检索和告警

### 6.2 链路追踪

**问题**：无法追踪一次用户请求经过了哪些环节、耗时多少。

**方案**：

```python
# utils/tracing.py（新增）
import uuid
import time
from contextvars import ContextVar
from utils.logger_handler import logger

trace_id: ContextVar[str] = ContextVar("trace_id", default="")

class Tracer:
    def __init__(self, name: str):
        self.name = name
        self.trace_id = trace_id.get("")
        self.start = None

    def __enter__(self):
        self.start = time.perf_counter()
        logger.info(f"[trace:{self.trace_id}] {self.name} 开始")
        return self

    def __exit__(self, *args):
        cost = (time.perf_counter() - self.start) * 1000
        logger.info(f"[trace:{self.trace_id}] {self.name} 完成, 耗时 {cost:.1f}ms")

def new_trace() -> str:
    tid = uuid.uuid4().hex[:12]
    trace_id.set(tid)
    return tid
```

```python
# app.py 中设置 trace_id
if prompt := st.chat_input():
    new_trace()  # 每次请求生成 trace_id
    ...
```

**收益**：全链路可追踪、耗时可度量、问题可定位

### 6.3 关键指标监控

**问题**：无法知道系统运行状态（QPS、延迟、错误率、token 消耗）。

**方案**：

```python
# utils/metrics.py（新增）
from collections import defaultdict
import threading
import time

class Metrics:
    _lock = threading.Lock()
    _counters = defaultdict(int)
    _timers = defaultdict(list)

    @classmethod
    def inc(cls, name: str, value: int = 1):
        with cls._lock:
            cls._counters[name] += value

    @classmethod
    def observe(cls, name: str, ms: float):
        with cls._lock:
            cls._timers[name].append(ms)
            if len(cls._timers[name]) > 1000:  # 滑动窗口
                cls._timers[name] = cls._timers[name][-500:]

    @classmethod
    def snapshot(cls) -> dict:
        with cls._lock:
            result = {}
            for k, v in cls._counters.items():
                result[k] = v
            for k, v in cls._timers.items():
                if v:
                    result[f"{k}_avg_ms"] = sum(v) / len(v)
                    result[f"{k}_p95_ms"] = sorted(v)[int(len(v) * 0.95)]
            return result
```

**指标设计**：

| 指标名 | 类型 | 含义 |
|-------|------|------|
| `request_total` | Counter | 总请求数 |
| `request_error` | Counter | 错误请求数 |
| `tool_call_total` | Counter | 工具调用次数（按工具名分标签） |
| `llm_latency` | Timer | LLM 调用延迟 |
| `retriever_latency` | Timer | 检索延迟 |
| `token_input` | Counter | 输入 token 数 |
| `token_output` | Counter | 输出 token 数 |

**收益**：运行状态可视化、性能瓶颈可量化
